1. How do you check the current status of your Git repository?

To check the current status of your Git repository, you can use the following command:

git status

This command will give you a summary of:

  • Changes in your working directory that haven’t been staged yet.
  • Changes that have been staged but not yet committed.
  • Files that are untracked (not added to the staging area).
  • Which branch you're currently on.

2. What is the difference between git fetch and git pull?

git fetch:

  • git fetch retrieves changes from the remote repository but doesn’t apply them to your working directory. It only updates your local references to the remote branches.
  • It allows you to see what changes are available before merging them into your current branch.
  • Use case: If you want to see the changes in the remote without actually modifying your working directory.
git fetch origin

git pull:

  • git pull is a combination of git fetch and git merge. It fetches the changes from the remote repository and immediately tries to merge them into your current branch.
  • Use case: If you want to bring your local branch up to date with the remote and automatically merge the changes.
git pull origin main

Summary: git fetch only downloads changes but doesn’t modify your working directory. git pull downloads and merges changes into your current branch.

3. How do you undo the last commit but keep the changes in your working directory?

To undo the last commit while keeping the changes in your working directory (so you can re-commit them), you can use:

git reset --soft HEAD~1
  • --soft: This option undoes the commit, but leaves the changes in the staging area (index).
  • HEAD~1: Refers to the commit before the current one. You are telling Git to reset to the commit just before your latest one.

Now, your changes will remain in your working directory and be staged, ready for you to make any necessary edits or re-commit.

Deleting Branches in Git

Delete a Local Branch

Use the following command:

git branch -d branch-name

Delete a Remote Branch

To delete a branch from the remote repository, use:

git push origin --delete branch-name

Handling Multiple People Working on the Same File in Git

Handling multiple people working on the same file in Git involves good practices to minimize conflicts and ensure smooth collaboration. Here are some steps and tips:

1. Communicate

Keep the team informed about changes being made. Use a messaging platform or issue tracker to discuss significant modifications.

2. Pull Regularly

Frequently pull the latest changes from the remote repository to stay updated with others' work.

git pull origin main

3. Use Feature Branches

Each person should work on their own feature branch rather than directly on the main branch. This reduces the chance of conflicts.

git checkout -b feature-branch

4. Commit Often

Make small, frequent commits to save your progress. This makes it easier to resolve conflicts when they occur.

5. Resolve Conflicts

If two people modify the same part of the file, Git will flag a conflict. Resolve it manually by editing the file to incorporate changes from both parties.

After resolving conflicts, stage the changes:

git add <file-name>

Then, complete the merge:

git commit

6. Use Pull Requests

If using platforms like GitHub, create pull requests for changes. This allows others to review and discuss modifications before they are merged into the main branch.

The Purpose of `git stash`

git stash is a command that allows you to temporarily save changes that you’re not ready to commit yet. It helps you create a clean working directory without losing your changes. Here’s when and why you might use it:

Purpose:

  • To save uncommitted changes in your working directory and index (staged changes).
  • To switch branches or pull new changes without committing your current changes.

When to Use:

  • If you are in the middle of working on a feature and need to quickly switch to another branch to address an urgent issue.
  • When you want to pull the latest changes from the remote repository but have uncommitted work.

Example Usage:

Stash Changes:

git stash

Switch Branches:

git checkout another-branch

Apply Stashed Changes:

After addressing the issue, you can switch back to your original branch and apply the stashed changes:

git checkout your-branch

Summary

Use git stash to temporarily store uncommitted changes, allowing you to work on something else without losing your progress.
