Why Docker is Useful

Docker helps in running applications in isolated environments called containers. Containers ensure that an app works the same way across different platforms, avoiding issues caused by system differences (software versions, libraries, or OS types).

How Docker Works

Container

A container is like a box containing your app and all its dependencies. It’s isolated from the system it runs on, ensuring no interference with other applications.

Image

An image is a template for creating containers. It includes everything the app needs to run, from software to configurations.

Dockerfile

The Dockerfile is a set of instructions used to create an image. It specifies the environment and dependencies needed for your app.

Docker Hub

Docker Hub is a platform for sharing and downloading images. You can share your app's image with others or use images shared by the community.

Docker Workflow

1. Build

Create a Dockerfile that describes the environment and setup needed for your app. Docker reads it and builds an image.

2. Run

Once the image is created, you can run containers from it. The app inside the container will be isolated and won’t affect others.

3. Share

Share your images with others via Docker Hub, or download images built by others to run on your own system.

4. Scale

When more users need access to your app, you can quickly launch multiple containers using the same image to handle the load.

Example

For example, if you're building a website:

  • You can package the website along with the required version of Python and libraries into a Docker container.
  • Your team can run it on their computers without worrying about setting up dependencies.
  • You can deploy the website on any server, and it will behave exactly the same as it did on your local machine.

Docker simplifies app management across different environments, ensuring consistency and making it easy to deploy, share, and scale applications.

Steps to Write a Dockerfile for a Spring Boot Application

1. Build Your Spring Boot Application

Before writing a Dockerfile, make sure your Java Spring Boot application is ready and packaged as a .jar file. You can package it using Maven or Gradle.

For Maven:

mvn clean package

This will create a .jar file in the target/ folder.

2. Basic Dockerfile for Spring Boot

Create a file named Dockerfile in the root of your Spring Boot project. Here’s a simple Dockerfile:


 FROM openjdk:17-jdk-alpine
 WORKDIR /app
 COPY target/my-spring-boot-app.jar app.jar
 EXPOSE 8080
 ENTRYPOINT ["java", "-jar", "app.jar"]

Explanation of Dockerfile Commands:

  • FROM openjdk:17-jdk-alpine: Uses a lightweight version of Java 17 as the base image.
  • WORKDIR /app: Sets the working directory inside the container.
  • COPY target/my-spring-boot-app.jar app.jar: Copies the built .jar file from your local machine to the container.
  • EXPOSE 8080: Opens port 8080 in the container for your Spring Boot app.
  • ENTRYPOINT: Runs the app using java -jar app.jar.

3. Build the Docker Image

Once the Dockerfile is ready, build your Docker image with the following command:

docker build -t my-spring-boot-app .

4. Run the Docker Container

Run the built Docker image with the following command:

docker run -p 8080:8080 my-spring-boot-app

Access your Spring Boot app at http://localhost:8080.

5. (Optional) Push Your Image to Docker Hub

If you want to share your Docker image, follow these steps:

  1. docker login
  2. Tag your image: docker tag my-spring-boot-app your-dockerhub-username/my-spring-boot-app
  3. Push your image: docker push your-dockerhub-username/my-spring-boot-app

Example Directory Structure


 my-spring-boot-app/
 ├── src/
 │   └── (Java source files)
 ├── target/
 │   └── my-spring-boot-app.jar
 ├── Dockerfile
 ├── pom.xml (or build.gradle)
 └── README.md

Summary of Steps:

  • Write Dockerfile: Define how Docker should package your Spring Boot app.
  • Build Docker Image: Use docker build to create an image.
  • Run Container: Use docker run to run your app in a container.
  • Share: Optionally, push your image to Docker Hub for sharing or deployment.
